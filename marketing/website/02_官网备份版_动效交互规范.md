# 官网备份版｜动效与交互规范（Interactions）

> 原则：动效服务于“可扫描与理解”，不是装饰；以 CSS 为主、少量 JS 触发；性能优先。

---

## 1) 页面级动效

- **背景光晕**：Hero 背景使用低频“呼吸”光晕（opacity/blur 轻微变化）
- **滚动渐显**：所有卡片/分区进入视口时 fade+slide（8–16px），统一节奏
- **数字递增**（可选）：若有“批量/复用/沉淀”等指标，用短动画提升可信度（避免夸大）

---

## 2) 组件级交互

### Feature Cards
- Hover：轻抬升（translateY -4px）+ 边框高亮（强调色 20% 透明）
- 点击：跳转到对应 section（或展开细节）

### Workflow Stepper（闭环四步）
- 默认选中“生成”
- 滚动到 stepper 区域后：随滚动高亮步骤（IntersectionObserver）
- 点击步骤：右侧内容切换（图片/文案/要点）

### Compare Table
- 行 hover：浅高亮
- 支持“只看差异”开关（可选）

### FAQ Accordion
- 默认折叠
- 展开动画：height 过渡 + icon 旋转
- FAQ 内容保持“中性口径”，避免负面情绪化措辞

---

## 3) 动效实现建议（技术）

- **滚动渐显**：IntersectionObserver + 添加 `is-visible` class
- **避免重布局**：尽量用 opacity/transform，不用频繁修改 height（FAQ 例外）
- **减少依赖**：不引入框架；如要渲染 Mermaid，建议预渲染 SVG 再嵌入
