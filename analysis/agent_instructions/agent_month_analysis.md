# 月度对话分析 Agent 指令

## 你的任务
分析指定月份的所有对话文件，从 6 个视角提取信息。

## 6 个分析视角

### 视角 1：时间线与节奏
- 识别密集期（同一天多个对话 = 可能卡住了）
- 识别空白期（连续几天没对话 = 放弃/独立探索/休息）
- 记录对话长度分布（特别长的对话往往是问题复杂）

### 视角 2：问题生命周期
- 找出跨会话出现的同一问题
- 标记"以为解决了但又冒出来"的问题
- 追踪问题的完整路径：出现→尝试→解决/放弃

### 视角 3：用户思维模式
- 分析提问方式（模糊 vs 精准）
- 识别表达习惯和常用词汇
- 评估用户直觉的准确率（用户判断对了几次）

### 视角 4：情绪与决策
- 标记挫败点（反复失败、情绪词汇）
- 标记突破点（问题解决、积极词汇）
- 识别妥协/将就的决策（"先这样"、"以后再说"）
- 记录放弃的问题（没有真正解决，绕过去了）

### 视角 5：工具与技术演进
- 记录 Claude/Codex/Serena 的使用情况
- 识别工具切换的时机和原因
- 追踪技术栈变化（PyQt5→PySide6, CSV→SQLite 等）

### 视角 6：知识盲区
- 找出用户不知道但本可以用的概念/工具
- 识别绕远路的解决方案
- 标记"后来才学到的更好方法"

## 输出格式
严格按照 output_schema.md 的 JSON 格式输出，写入到 analysis/overnight/month_YYYY_MM.json

## 注意事项
- 每个发现都要附带原始对话的引用（文件名 + 关键片段）
- 不要泛泛而谈，要有具体例子
- 宁可漏掉不确定的，也不要瞎编
- 用中文输出所有分析内容
