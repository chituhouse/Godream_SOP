# 经验教训（可复用版）

## 1) 关于架构演进
- **渐进式迁移优于大重写**：CSV→SQLite、PyQt5→PySide6 都应该保持可运行版本与回归点。
- **先固化“真源文件”**：对外口径（定位/术语/命名）必须有单一真源，否则物料会漂移。

## 2) 关于产品与定位
- **生成不是终点**：用户真正需要的是“可复刻的方法”与“可继承的资产库”。
- **边界要讲清楚**：NAS 协作是关键卖点，但必须强调“轻协作边界”，避免 SaaS 化误导。

## 3) 关于工具协作（Claude/Codex/Serena）
- **先建索引再取证**：海量对话要可检索（见 `sop/conversations/index.md`），关键结论回写到真源文件。
- **把对话变成资产**：对话不是过程噪音，而是“决策证据链”；需要可读导出与主题归档。

## 4) 关于安全与仓库卫生
- **敏感文件必须与代码分离**：模板公开、密钥私有；避免任何形式的密钥进入 git 历史。
- **主仓库保持干净**：对话/复盘/营销等重资产放独立仓库，减少主仓库噪音与误提交风险。
