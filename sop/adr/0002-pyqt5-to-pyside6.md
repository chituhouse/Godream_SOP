# ADR 0002: PyQt5 → PySide6（UI 框架迁移）

## 状态
- 接受（Accepted）

## 背景（Context）
- 早期 UI 基于 PyQt5 搭建，后期需要更现代的 Qt6 生态与长期维护空间。
- 项目进入“桌面端产品化”阶段，需要更可持续的 UI 架构与打包发布流程。

## 决策（Decision）
- 迁移到 PySide6，并逐步完成 UI 组件与主题系统的现代化（以实际实现为准）。

## 影响（Consequences）
### 正面
- 更现代的 API/生态，利于长期维护与跨平台打包。
- 为后续 UI 组件化、主题 token 化、交互一致性奠定基础。
### 负面 / 风险
- 迁移期风险：UI 行为差异、兼容问题、回归成本上升。
### 缓解措施
- 分阶段迁移与验收：每个阶段有可运行版本与回归点（以实际实现为准）。

## 备选方案（Alternatives）
- 继续 PyQt5：短期稳定，但长期维护与生态演进空间受限。
- 重写为其他前端技术栈：成本过高且风险不可控。
