# Phase 4–7 最终验收清单 + 物料结构（GoDream_SOP）

> 目标：把 `gemeng2` 的研发资产（代码/文档/对话）沉淀为一套可复用 SOP + Marketing 物料体系，便于“火山销售转发即用”。
>
> 对外口径真源：`marketing/00_对外口径真源.md`

---

## Phase 4｜官网备份版（文案 + 结构 + 图谱 + 动效规范）

1. **信息架构完成**：官网分区（Hero/痛点/闭环/功能/工作流/协作/对比/FAQ/CTA）明确且不自相矛盾。
2. **定位一致**：全站统一为“**AIGC 视觉资产管理工具（桌面端）**”，不写“平台/自研模型/公有云 SaaS”。
3. **价值主张清晰**：主叙事是“生成 → 存储 → 复用 → 交付/轻协作”，生成能力不喧宾夺主但必须可感知。
4. **图文并茂**：至少 6 张可复用图谱（Mermaid/线稿/思维导图均可）覆盖：闭环、图片工作流、视频工作流、NAS 协作、数据沉淀、选型对比。
5. **高端视觉规范**：深色/留白/字体/层级/图标风格（线性/等线）形成可交付的 style guide。
6. **动效规范**：列出每个 section 的动效意图（进入/滚动/悬停/数值递增/高亮流动）与实现建议（纯 CSS/轻 JS）。
7. **“证据链”补齐**：把能支撑卖点的截图清单列出来（由你后续配图），并标注每张图的文案点位。
8. **FAQ 策略落地**：性能/速度/限制类只进 FAQ（如“组图约 1 分钟/张”），不出现在 Hero/主卖点。

## Phase 5｜公众号体系（模板 + 文章集）

### Phase 5.0｜模板与一致性（先锁口径）
1. **公众号模板**：统一结构（标题/导语/痛点/方案/步骤/配图位/FAQ/CTA）可复用。
2. **术语表固化**：模型命名统一 `Doubao Seedream 4.5` / `Doubao Seedance 1.5 Pro` 等，写“接入/调用”不写“自研”。
3. **CTA 统一**：咨询路径（企业微信/火山经理）与“转发给客户即可读懂”写法固定。
4. **语气分层**：发布稿偏故事与大局；功能稿偏专业与可落地；场景稿偏轻松与代入。

### Phase 5.1｜发布稿 + 核心功能稿（第一批）
1. **发布稿 1 篇**：让读者 3 分钟内知道“鸽梦是什么/给谁用/解决什么/下一步怎么试用”。
2. **功能稿 ≥3 篇**：至少覆盖图片工作流、视频工作流、资产沉淀/复用闭环。
3. **每篇都有配图清单**：每个小节明确需要什么截图/示意图（由你后续补图）。

### Phase 5.2｜其余功能稿（第二批，补齐系统性）
1. **功能稿补齐到 6–7 篇**：批量/组图、多模态参考、NAS 私有云协作、迁移与继承、对比与选型等。
2. **FAQ 不抢叙事**：速度/限制/适用团队规模等全部沉到底部 FAQ。

### Phase 5.3｜场景稿（第三批，5 篇）
1. **场景稿 5 篇齐备**：每篇按“角色—任务—痛点—工作流—收益—可复用模板”写。
2. **可转发性验收**：火山销售转发给客户，客户不需要再追问“这产品到底干什么”的基础问题。

## Phase 6｜销售资料包（火山销售转发即用）

1. **One-pager（1 页）**：一句话定位 + 3 卖点 + 适用人群 + 1 张闭环图 + CTA。
2. **5 分钟 Pitch**：面向销售快速讲清楚“是什么/为何现在/为何我们/怎么开始”。
3. **15 分钟 Pitch**：加上“对比选型 + 部署/使用边界 + FAQ”。
4. **企业微信自动回复**：首次加好友、二次跟进、预约演示、试用转化四套话术。
5. **异议处理**：围绕“为什么不用即梦/为什么不用 Eagle/为什么不用平台”给出标准答案。
6. **火山销售培训稿**：把“定位口径/可讲点/不可讲点/演示路径/常见问题”做成脚本。

---

## Phase 7｜复盘与知识库（可长期演化）

1. **双索引**：按时间线 + 按主题（产品/技术/包装/营销/协作/定价）可检索。
2. **关键决策（ADR）模板**：每次重大迁移（CSV→SQLite、PyQt5→PySide6）都有“为什么/权衡/结果”。
3. **月度复盘模板**：目标-进展-问题-改进-下月计划。
4. **对话取证机制**：先查索引再回看原始对话，关键结论回写到“真源文件”防漂移。
5. **里程碑补齐**：版本演化（v0.x→v1.x）按阶段列清楚“功能/架构/工具链”的变化。

## 公众号文章清单（标题 + 大纲）

> 说明：以下是“火山销售转发即用”的基础文章集；每篇都预留“配图建议”和“FAQ/边界”。

### 00｜发布稿（1 篇）
**《鸽梦 GoDream 正式发布：让 AIGC 产出变成可复用的视觉资产库》**
- 你是谁（不是平台、不是自研模型）：桌面端 AIGC 视觉资产管理工具
- 你解决什么：从“生成后散落”到“生成即沉淀、随时复用”
- 你怎么做：接入火山 Doubao-Seed 模型 → 统一工作流 → 本地/NAS 资产库
- 适合谁：电商/短视频/品牌内容团队中“负责 AI 生成的人”
- 30 秒理解闭环图（配图位）
- CTA：试用/咨询/对接火山渠道

### 功能稿（建议 6 篇）
**01｜《图片工作流：接入 Doubao Seedream 4.5，把“批量出图/套图一致性”变成日常能力》**
- 场景：电商主图/详情页套图/系列海报
- 三种模式：单张、批量、组图/套图（卖点在“一次出一套”）
- 多模态参考：上传参考图 → 理解/对齐风格 → 迭代
- 资产沉淀：每张图的 Prompt/Seed/参考图/参数自动入库
- FAQ：组图速度与等待策略（仅放底部）

**02｜《视频工作流：Doubao Seedance 1.5 Pro 在桌面端的可控生成（含首尾帧）》**
- 你能得到什么：更可控的视频生成与可追溯参数
- 典型流程：文生视频 / 图生视频 / 首尾帧
- 结果管理：缩略图、回看、复用同一套参数继续迭代
- FAQ：时长/分辨率/生成耗时/失败重试

**03｜《生成即资产：为什么“实时存储 + 全量元数据”比多跑 100 张图更重要》**
- 痛点：神图找不到、参数丢失、复刻靠运气
- 机制：结果 + Prompt/参数/参考图/Seed 同步沉淀
- 你能做什么：检索、筛选、对比、回滚
- 配图：一张“资产卡片”信息结构示意

**04｜《一键复用：把一次成功变成可复刻模板（从神图到系列化产出）》**
- 从历史到复刻：调用记录 → 改 1 个参数 → 批量变体
- 为什么更稳：参数追踪（requested/sent/actual）与可回溯
- 配图：复用入口、变体产出、对比视图

**05｜《私有云协作（NAS）：鸽梦如何让团队共用一个 AI 资产库（数据不出公司）》**
- 原理图：相对路径 + NAS 映射 = 多人看到同一项目
- 边界：更适合 3–10 人小团队；并发写入的流程建议
- 配图：NAS 映射示意 + 共享资产库结构

**06｜《为什么不是另一个“生图工具”：鸽梦与生成平台 / Eagle / 文件夹的差异》**
- 生成平台：强生成弱沉淀；迁移与复用成本高
- Eagle/文件夹：强管理弱生成；参数/seed 不可追溯
- 鸽梦：把“生成 + 沉淀 + 复用”做成闭环
- 一张对比表（配图位）

### 场景稿（5 篇）
**S1｜《电商设计：主图/详情页/套图一次出，沉淀成可复用素材库》**
- 角色：电商设计师/运营
- 任务：主图、详情页套图、节日活动海报
- 工作流：参考图 → 组图 → 筛选 → 复用 → 批量迭代
- 收益：交付更快、风格更统一、素材可继承

**S2｜《短视频团队：封面与素材的“高频小迭代”，不再从零开始》**
- 角色：短视频编导/剪辑/封面设计
- 任务：封面、片头片尾、关键镜头素材
- 工作流：Seedance 生成 → 入库 → 复用参数做系列
- 收益：系列化更稳定、沟通成本下降

**S3｜《品牌营销：Campaign 资产从“散落的图”变成“可追溯的资产包”》**
- 角色：品牌市场/设计
- 任务：一套 Campaign 物料（KV、海报、短视频）
- 工作流：统一风格参考 → 批量生成 → 资产包交付
- 收益：一致性、复刻性、交付可追溯

**S4｜《影视/动画：概念设定与分镜的快速探索 + 可复用资产沉淀》**
- 角色：概念设计/分镜师
- 任务：角色/场景/镜头参考
- 工作流：参考图 → 迭代 → 入库 → 版本对比
- 收益：探索更快、版本更清晰、交接更顺畅

**S5｜《游戏/泛娱乐：角色/道具/场景的系列化产出与素材管理》**
- 角色：美术/策划
- 任务：设定稿、素材库、宣传图
- 工作流：组图/批量 → 筛选 → 复用 → 系列化
- 收益：素材沉淀、系列统一、可迁移可继承
